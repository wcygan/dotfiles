Research the topic specified in the command arguments using web search.

**Research process:**
1. **Search**: Query the web for relevant, up-to-date information
2. **Synthesize**: Combine findings from multiple sources
3. **Validate**: Cross-reference information for accuracy
4. **Contextualize**: Relate findings to development/engineering context

**Output format:**

# Research: {topic}

## Key Findings
- **Main insight 1**: Brief explanation with source
- **Main insight 2**: Brief explanation with source
- **Main insight 3**: Brief explanation with source

## Detailed Analysis
{Comprehensive explanation of the topic based on search results}

## Relevant Examples
{Code snippets, configuration examples, or practical applications if applicable}

## Best Practices
{Industry standards, recommendations, common patterns}

## Gotchas & Caveats
{Common pitfalls, limitations, version-specific issues}

## Sources
- [Source 1 title](url)
- [Source 2 title](url)
- [Source 3 title](url)

**Research focus areas:**
- Official documentation (prioritize)
- Recent blog posts and tutorials (prefer last 2 years)
- GitHub repositories and discussions
- Stack Overflow solutions
- Technical specifications

**Quality standards:**
- Verify information against multiple sources
- Prefer official docs over third-party when available
- Note version-specific details (e.g., "as of v2.0")
- Flag deprecated or outdated practices
- Include publication dates when relevant

**Usage examples:**
- `/web-research Rust async best practices 2025`
- `/web-research Kubernetes ingress controller comparison`
- `/web-research PostgreSQL connection pooling strategies`

Perform web research on: $ARGUMENTS
